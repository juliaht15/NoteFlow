## Flujo de Trabajo
Para mantener el orden durante el desarrollo, las tareas se mueven a través de las siguientes columnas:

*   **Backlog**: Funcionalidades planeadas y tareas técnicas pendientes de priorización.
*   **Todo**: Tareas seleccionadas para el sprint actual.
*   **In Progress**: Funcionalidad en desarrollo activo.
*   **Review**: Código terminado esperando revisión o comprobación de errores.
*   **Done**: Tareas completadas, probadas y funcionales en el entorno de desarrollo.

## Notas de Gestión
*   Cada funcionalidad principal (Notas, Checklists, Ideas) cuenta con una tarjeta dedicada en Trello.
*   Las subtareas técnicas (ej. "Configurar tipado en FlashList", "Implementar middleware de persistencia") se desglosan en checklists dentro de cada tarjeta.
*   Se recomienda realizar commits atómicos referenciando el ID de la tarjeta de Trello.
